### 步骤 1: 构造搜索查询 (按天进行)
对于每一天 (`current_date`)，使用以下查询获取最多 1000 个结果：
`created:{current_date} size:50..80000 pushed:>{current_date_plus_1day} stars:0..3000`
* *解释*：筛选 50KB-80MB 大小、创建后有提交记录（非僵尸号）、且非超大型机构的项目。

### 步骤 2: 本地过滤与分层抽样 (处理每个 Repo)
对 API 返回的每个仓库，按顺序执行以下逻辑：

**A. 负向关键词过滤 (去噪)**
如果 `description` 或 `topics` 包含以下任一词汇（忽略大小写），则**跳过**：
* `homework`, `assignment`, `course`, `tutorial`, `demo only`, `learning`, `study`
* `deprecated`, `archived`, `backup`, `dotfiles`, `config`
* `leetcode`, `algorithm`, `solutions`

**B. 分层抽样 (质量控制)**
* **Tier 1 (沉默的大多数)**: 如果 `stars <= 20`：
    * **随机保留 20%**。
    * 逻辑：`if random.random() > 0.2: continue`
    * 在输出中标记：`"tier": "silent"`
* **Tier 2 (高价值信号)**: 如果 `stars > 20`：
    * **100% 保留**。
    * 在输出中标记：`"tier": "signal"`

### 步骤 3: 获取 README (注入灵魂)
* 对于通过上述筛选的仓库，调用 `repo.get_readme()`。
* **必须解码**：将 Base64 内容解码为 UTF-8 字符串。
* **容错处理**：使用 `try-except` 包裹。如果 404 (无 Readme) 或其他网络错误，将 `readme_content` 设为 `null`，**绝对不要让脚本崩溃**。

### 步骤 4: 流式落盘
* 每处理完一个仓库，**立即**追加写入到 `vibe_coding_dataset_2w.jsonl` 文件中。
* **保存字段**：
