# Vibe Coding 核心分析框架

> 核心问题：Vibe coders 们在做什么？趋势是否在加速？

---

## 分析目标

1. **现状洞察**：Vibe coders 主要在构建什么类型的项目？
2. **趋势判断**：过去两周，Vibe Coding 是在升温还是降温？

---

## 模块一：Vibe Coders 在做什么

### 1.1 基础分布（已有）

| 维度 | 目的 |
|------|------|
| 宏观类别分布 | 个人工具 vs 基础设施 vs 产品原型 |
| 微观场景分布 | 12 个具体赛道的项目数量 |
| AI 参与度分布 | 1-5 分的占比（看 AI 渗透程度）|
| 复杂度分布 | 项目技术难度分布 |

### 1.2 关键交叉分析

```
问题 1：AI 参与度与项目类型的关系
- 个人工具 / 基础设施 / 产品原型，哪类 AI 浓度最高？
- 输出：三类平均 AI 分数对比柱状图

问题 2：AI 参与度与复杂度的关系
- 假设：AI 让开发者敢做更复杂的项目
- 验证：ai_generation_score vs complexity_level 的散点图

问题 3：哪些场景最依赖 AI？
- 12 个微观场景各自的平均 AI 分数排名
- 输出：场景 AI 浓度排行榜
```

### 1.3 项目意图词频（轻量级）

```
方法：提取所有 core_intent，分词统计高频动词
输出：Top 20 动词（如"构建"、"管理"、"生成"、"分析"）
价值：一眼看出 vibe coders 的行为模式
```

## 模块二：时间序列分析（重点）

### 2.1 项目创建趋势

```
数据来源：vibe_coding_dataset_2w.jsonl 中的 created_at

分析维度：
1. 每日新创建仓库数量（原始量）
2. 每日通过过滤的仓库数量（有效量）
3. 每日进入分析样本的仓库数量（抽样后）

输出图表：
- 时间序列折线图（3 条线）
- 7 日移动平均线（平滑噪声）
- 关键指标：日环比增长率、周复合增长率
```

### 2.2 AI 浓度趋势

```
分析维度：
- 每日平均 ai_generation_score 变化
- 高 AI 项目（>=4 分）占比变化
- 极高 AI 项目（5 分）占比变化

洞察价值：
- AI 工具的使用是在加深还是停滞？
- 开发者对 AI 的依赖度是否在提升？
```

### 2.3 场景热度趋势

```
分析维度：
- 每日各 micro_scenario 的新项目数量
- 场景占比的 7 日滑动平均

输出：
- 场景热度堆叠面积图（看结构变化）
- 新兴场景识别（连续增长）
- 衰退场景识别（连续下降）
```

### 2.4 质量趋势

```
分析维度：
- 每日平均 stars（社区认可度）
- 每日平均 complexity_level（技术复杂度）
- 每日进入 signal 层（>20 星）的项目数

洞察：
- Vibe Coding 是在"泡沫化"（量增质降）还是"进化"（量质齐升）？
```
